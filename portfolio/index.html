Adobe

Project Auguste

Product Design, UX, UI, ID, Development

Project Auguste, a touch-first interactive sculpting mobile app, is part of Adobe's Project Felix initiative.

Goal:
My task was to take technology and wrap a product design around it that would comply with and extend Adobe's application aesthetic. I was given four months to produce a demo app to be presented at Adobe MAX 2016.

Process:
Project Auguste's engine was designed by Adobe's Sebastian Marketsmueller and allows the user to sculpt in a virtual space, complete with various brushes, filters, and symmetrical drawing effects. I introduced an interaction system that would allow users to interact with the camera object while at the same time editing the object's surface using a combination of multi-touch gestures and optionally using a stylus.

Outcome:
We produced a Windows SurfaceBook app that aligned perfectly with Adobe's world-class products while extending the current input conventions allowing for a rich multitouch and touch-stylus gesture system to create amazing 3D objects in real-time.

Conclusion:
Although the product was very well-received, the post even data showed something else:

  • Creative professionals want to use tools like Z-Brush
  • The target market for the product was more artist/sculptural, as opposed to a creative hobbyist.
  • 3D is still a tough sell for creatives and non-creatives alike and this was too niche.

Streamlio

Direction, Design, Artwork, Development

Streamlio is a Palo Alto startup that provides unified solutions for real-time streaming applications and services that are easy to build, adapt, and scale reliably.

Goal:
They contacted me to refresh their logo, build out their branding guidelines, and to develop their web site to have a contemporary look and feel and make accessing information easier.

Process:
I separated the work into two main parts: Branding and Web Design/Development. Quick iterations and weekly meetings keep the project on track. I quickly developed the Branding Guidelines, then IA and wireframes for the site for review and testing. I followed up by developing the web style guide then implementing the web components.

Outcome:
With a 546% increase in web traffic, higher SEO ranking, better search/referral, and higher view rate of articles and videos, we succeeded in meeting our goals. Modular HTML/HUGO/SASS blocks provided a way to compose new web page layout and content for growing the site for future needs.

Conclusion:
It was a fun project and ran smoothly there were several issues in 'not-so-dry' SASS-HUGO blocks that required additional time in refactoring. During the work, I learned Hugo, a static web site generation tool, which I currently use in my web development projects.

CraftMaster Software

CraftMaster Software

Founder, Design Lead, Web/App Development

CraftMaster Software was started with a singular purpose, to create a tool for users of all ages to import, export, beautify, and share there Minecraft creations.

Goal:
The main goals to address:

  • Create a brand, web site, and marketing content.
  • Design a multi-platform application for users of all ages to import/export 3D objects to and from Minecraft.

Process:
I worked closely with the team to coordinate design, marketing, and engineering efforts. We used weekly meetings and design sprints to understand the main goals. I created corporate logos, branding guidelines, icons, and additional assets for web, marketing, and application use. I worked with Andy to understand the user base and market, then created personals and workflows. Working directly with Pete we quickly iterated thought a tech demo to product demo and started user testing and feedback. We iterated over the software developing the feature set for the final product.

Outcome:
CraftMaster not only allowed for 3D model import/export but also included several 3D repositories acting a unified 3D model library. CraftMaster also could share your 3D models to Facebook and other sites supporting the Shapeways 3D Viewer.

Conclusion:
Although CraftMaster has great potential, due to patent licensing issues the road to profitability was a dead-end and the project had to be canceled. However, the CraftMaster app is still being used by more than 1,000 users.
