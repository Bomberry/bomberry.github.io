Hide Desktop icons on macOS with AppleScript and Shell script

Using a combination of a Terminal command and an AppleScript, we can easily hide and show Desktop icons.

Tinkering with Terminal

You’ve been there. You want to take a screenshot, or record your screen, or are going to share your screen for a presentation, only to realize that you have too many icons and folders on your desktop.

You can go into your Finder preferences and hide everything, but still, there are icons on your desktop. Unsightly!

So, I did a little digging and learned there’s a Terminal shell command that instructs Finder to build, or not to build, the desktop when it launches. Then, you need to “kill” Finder to relaunch it.

Here’s how to hide the desktop icons. Open Terminal and type the following:

defaults write com.apple.finder CreateDesktop -bool false; killall Finder

Setting CreateDesktop to “false” tells Finder not to build, or “create,” the desktop icons. Next, we kill the Finder process, which relaunches it. Since the CreateDesktop flag is set to false, the desktop isn’t built and nothing shows on the desktop.

Here’s how you can show them:

defaults write com.apple.finder CreateDesktop -bool true; killall Finder

You probably noticed this is almost the same command as before. We changed the boolean variable to “true” so Finder rebuilds the desktop and shows the icons. Next, we kill Finder, forcing it to launch and build the icons.

Augmenting with AppleScript

Great! But, using Terminal is required. Not a bad thing, but not the best user experience, either.

How can we make this better?

Here’s where AppleScript, and the Script Editor app come in.

Open Script Editor, create a new script, and click the “title” to rename it, “Desktop - Hide Icons.” Now, type the following:

do shell script "defaults write com.apple.finder CreateDesktop -bool false; killall Finder"

Save the file. I save my scripts to iCloud’s Script Editor folder.

Now, in the “hide” script, change “-bool false” to “-bool true” then select File > Save as… and save the file as “Desktop - Show Icons.”

You now have two scripts waiting to be used, but how do you access the scripts easily? Script Editor, again.

Make it accessable

With Script Editor still open, go to Script Editor in the menu and select Preferences. In the General tab, towards the bottom, you’ll see Script Menu and a checkbox next to it with the label, “Show Script menu in the menu bar.” Check that so it’s “on.”

You’ll notice there’s a new AppleScript icon in your menu bar near the date/time. If you click on it, you’ll see one of two things, a short menu with two menu items, or, if you also have “Show Computer scripts” checked on, an additional list of folders of various scripts you can use. But, our scripts are not in the menu… yet.

To use your newly created scripts, you need to copy them from where you saved them and paste them into the Scripts folder.

In Finder, open a new window and navigate to iCloud Drive, then Script Editor. Select the two scripts and copy them, either with the keyboard shortcut or selecting Edit > Copy from the menu.

Next, click on the AppleScript icon in the menu and select Open Script Folder > Open User Scripts Folder. A new Finder window will open to the folder where your scripts are stored (~/Library/Scripts).

Paste the scripts into the folder by using the keyboard shortcut, Edit > Paste menu item, or if you have the iCloud Script Editor window still open, drag and drop them to the folder holding Option to make a copy.

Click on the AppleScript icon in the menu. You will see your scripts in the list and now have easy access to hide and show your desktop icons.

I hope that this helps you in your tasks. Happy recording, capturing, and screen sharing with a clutter-free desktop!
